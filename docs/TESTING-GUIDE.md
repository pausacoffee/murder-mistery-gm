# 테스트 가이드

## 목표
이 프로젝트의 테스트는 브라우저 UI 전체를 한 번에 검증하기보다, 먼저 계산 로직을 순수 함수로 분리해 빠르게 검증하는 것을 목표로 합니다.

## 현재 도구
- 테스트 러너: `Vitest`
- 순수 함수 모듈: `lib/pure-logic.js`
- 테스트 파일: `tests/pure-logic.test.mjs`

## 실행 방법
```bash
npm test
```

- 기본 `npm test`는 verbose 출력으로 실행됩니다.
- 테스트 이름은 한국어 문장으로 작성해, 터미널에서 어떤 규칙이 통과/실패했는지 바로 읽을 수 있게 유지합니다.

## 어떤 코드를 먼저 테스트하나요?
- 정렬
- 검색 필터
- 라우팅 해석
- 제목 포맷
- 메일 초안 생성
- 테마 경로 계산
- 헤더 액션 노출 범위 계산
- 문의 모달 단계 전환 규칙
- 렌더용 문자열 생성

## 순수 함수 후보 기준
다음 조건을 만족하면 우선 순수 함수 후보로 봅니다.

- 입력을 받으면 결과를 계산해 반환한다
- `document`, `window`, `Element`가 없어도 실행된다
- DOM 조작 없이 검증 가능하다

예:
- 게임 검색/정렬
- 해시 라우트 파싱
- 문의 메일 제목/본문 조합
- 작은 화면용 타이틀 줄바꿈 규칙

## 피해야 할 것
- 새 계산 로직을 바로 `render...()` 함수 안에 하드코딩
- DOM이 필요한 함수와 정렬/판단 로직을 한 함수에 섞기
- 테스트 가능한 계산을 굳이 `window.innerWidth`, `document.querySelector()`에 직접 묶기

## 추천 작업 순서
1. 새 기능의 계산 로직을 순수 함수로 분리
2. Vitest 테스트 추가
3. `app.js`에서 그 함수를 호출하도록 연결
4. `npm test` 통과 확인
5. 브라우저 수동 확인
