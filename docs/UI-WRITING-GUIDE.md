# 문구 / 표기 가이드

## 1. 공통 표기
- `Part` -> `파트`
- `Round` -> `라운드`
- 표기 형식은 `라운드 1`
- `1라운드` 같은 뒤치기 표기는 사용하지 않습니다.

## 2. 슬라이드 타이틀
- 파트 슬라이드는 `[파트 n] ...` 형식을 우선 사용합니다.
- 예: `[파트 2] 라운드 3: 탐색과 정보 공유`
- 작은 화면에서는 `:` 뒤에 줄바꿈이 들어갈 수 있으므로, 제목은 너무 길지 않게 유지합니다.

## 3. 버튼 문구
- 버튼은 동작 중심 문구를 사용합니다.
- 게임 특화 버튼은 공통 버튼으로 취급하지 않고, `headerActions[]` 데이터로 선언합니다.
- 예:
  - `이야기 카드 A 공개`
  - `메일 앱 열기`
  - `입장`

## 4. 확인 모달 문구
- 질문과 결과 안내를 분리합니다.
- 핵심 오브젝트는 먼저 제시합니다.
- 파트 전환 안내는 짧고 명확하게 씁니다.

## 5. 연락 모달 문구
- 문의 유형은 짧고 구분 가능해야 합니다.
- 권리자 요청 문구는 권한 확인이 필요하다는 점을 명확히 적습니다.
- 피드백 문구는 캡처, 재현 정보, 사용 환경을 함께 보내 달라고 안내합니다.
- `mailto:` 사용 시 메일 앱이 열리지 않을 수 있다는 안내를 포함합니다.

## 6. 순서 다이어그램 문구
- 흐름 노드에는 핵심 단계명만 넣습니다.
- `→`, `=>`, `->` 같은 텍스트 화살표는 사용하지 않습니다.
- 화살표는 CSS 또는 SVG 벡터 요소로 표현합니다.

## 7. 스토리 슬라이드 작성 규칙
- `스토리` 슬라이드는 원문 본문의 서사 흐름을 유지해 작성합니다.
- 임의 요약, 과도한 축약, 설명용 재서술만으로 대체하지 않습니다.
- 일본어 원문처럼 외국어 텍스트를 옮길 때는 직역투를 줄이고 자연스러운 한국어 문장으로 다듬되, 사건의 순서와 정보량은 유지합니다.
- 대사는 일반 서술과 섞지 말고 отдель한 문단으로 분리해 `story-dialogue` 스타일 적용 대상이 되게 작성합니다.
- 분위기를 끌어올리는 마지막 한두 문장, 장면 전환 문장, 챕터 마감 문장은 `story-final-line` 적용 대상인지 검토합니다.
- `story-slide`와 `scrollable-text` 구조를 우선 사용합니다.

## 8. 새 슬라이드 추가 시 체크리스트
1. 제목이 파트/라운드 표기 규칙에 맞는지 확인
2. `i` 버튼 연결이 필요하면 `data-part-info-topic` 또는 `data-part-info-title` 지정
3. 타이머면 `data-timer-widget` 계열 속성 확인
4. 순서 다이어그램이면 텍스트 화살표를 쓰지 않았는지 확인
5. 스토리 슬라이드라면 대사와 마지막 문장 스타일 적용 여부 확인
6. 작은 화면에서 제목 줄바꿈이 어색하지 않은지 확인
