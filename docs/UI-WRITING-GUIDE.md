# 문구 / 표기 가이드

## 1. 공통 표기
- `Part` -> `파트`
- `Round` -> `라운드`
- 표기 형식은 `라운드 1`
- `1라운드` 같은 뒤치기 표기는 사용하지 않습니다.

## 2. 슬라이드 타이틀
- 파트 슬라이드는 `[파트 n] ...` 형식을 우선 사용합니다.
- 예: `[파트 2] 라운드 3: 탐색과 정보 공유`
- 작은 화면에서는 `:` 뒤에 줄바꿈이 들어갈 수 있으므로, 제목은 너무 길지 않게 유지합니다.

## 3. 버튼 문구
- 버튼은 동작 중심 문구를 사용합니다.
- 게임 특화 버튼은 공통 버튼으로 취급하지 않고, `headerActions[]` 데이터로 선언합니다.
- 예:
  - `이야기 카드 A 공개`
  - `메일 앱 열기`
  - `입장`

## 4. 확인 모달 문구
- 질문과 결과 안내를 분리합니다.
- 핵심 오브젝트는 먼저 제시합니다.
- 파트 전환 안내는 짧고 명확하게 씁니다.

## 5. 연락 모달 문구
- 문의 유형은 짧고 구분 가능해야 합니다.
- 권리자 요청 문구는 권한 확인이 필요하다는 점을 명확히 적습니다.
- 피드백 문구는 캡처, 재현 정보, 사용 환경을 함께 보내 달라고 안내합니다.
- `mailto:` 사용 시 메일 앱이 열리지 않을 수 있다는 안내를 포함합니다.

## 6. 순서 다이어그램 문구
- 흐름 노드에는 핵심 단계명만 넣습니다.
- `→`, `=>`, `->` 같은 텍스트 화살표는 사용하지 않습니다.
- 화살표는 CSS 또는 SVG 벡터 요소로 표현합니다.

## 7. 새 슬라이드 추가 시 체크리스트
1. 제목이 파트/라운드 표기 규칙에 맞는지 확인
2. `i` 버튼 연결이 필요하면 `data-part-info-topic` 또는 `data-part-info-title` 지정
3. 타이머면 `data-timer-widget` 계열 속성 확인
4. 순서 다이어그램이면 텍스트 화살표를 쓰지 않았는지 확인
5. 작은 화면에서 제목 줄바꿈이 어색하지 않은지 확인
